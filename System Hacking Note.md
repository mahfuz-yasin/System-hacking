# 🛡️ System Hacking & Password Cracking: Professional Guide

সিস্টেম হ্যাকিং কোনো একক কাজ নয়, বরং এটি একটি সুশৃঙ্খল প্রক্রিয়া। নিচে **CEH (Certified Ethical Hacker)** স্ট্যান্ডার্ড অনুযায়ী এর ধাপ এবং কৌশলগুলো সাজানো হলো।

---

## 📂 সেকশন ১: সিস্টেম হ্যাকিংয়ের ৫টি প্রধান ধাপ
সিস্টেম হ্যাকিংয়ের মূল লক্ষ্য হলো একটি ডিভাইসের পূর্ণ নিয়ন্ত্রণ নেওয়া।

| ধাপ | লক্ষ্য (Goal) | টেকনিক (Technique) |
| :--- | :--- | :--- |
| **১. Gaining Access** | সিকিউরিটি বাইপাস করে সিস্টেমে ঢোকা। | Password Cracking, Exploitation, Social Engineering. |
| **২. Escalating Privileges** | সাধারণ ইউজার থেকে Admin/System হওয়া। | Exploiting system vulnerabilities & misconfigurations. |
| **৩. Executing Apps** | সিস্টেমে স্থায়ী এক্সেস বা রিমোট কন্ট্রোল নেওয়া। | Trojans, Backdoors, Keyloggers, Spyware. |
| **৪. Hiding Files** | নিজের ফাইল ও কাজগুলো লুকিয়ে রাখা। | Rootkits, Steganography. |
| **৫. Covering Tracks** | হ্যাকিংয়ের ডিজিটাল প্রমাণ মুছে ফেলা। | Clearing Log files (Event logs, Bash history). |

## 🔑 সেকশন ২: পাসওয়ার্ড ক্র্যাকিং মেথডোলজি
পাসওয়ার্ড ক্র্যাকিং হলো সিস্টেমে প্রবেশের সবচেয়ে কমন রাস্তা। এটি প্রধানত দুইভাবে হয়:

### ক. অনলাইন অ্যাটাক (Active Online Attacks)
1. **Dictionary Attack:** একটি নির্দিষ্ট ফাইল (যেমন: `rockyou.txt`) ব্যবহার করে সব কমন পাসওয়ার্ড চেক করা।
2. **Brute-Force Attack:** অক্ষরের প্রতিটি সম্ভাব্য কম্বিনেশন ট্রাই করা। এটি অনেক সময়সাপেক্ষ।
3. **Rule-based Attack:** যখন পাসওয়ার্ডের প্যাটার্ন সম্পর্কে ধারণা থাকে (যেমন: নামের সাথে জন্ম সাল), তখন নির্দিষ্ট নিয়মে অ্যাটাক চালানো।

### খ. নন-ইলেকট্রনিক অ্যাটাক (Physical/Social)
* **Social Engineering:** মানুষকে কৌশলে ভুলিয়ে তথ্য সংগ্রহ।
* **Shoulder Surfing:** সরাসরি কিবোর্ডের দিকে তাকিয়ে পাসওয়ার্ড দেখে নেওয়া।
* **Dumpster Diving:** ময়লার ঝুড়ি থেকে ফেলে দেওয়া গুরুত্বপূর্ণ কাগজ বা নোট উদ্ধার করা।

## 🛠️ সেকশন ৩: প্রাকটিক্যাল টুলস এবং রিসোর্স
পেনটেস্টিং বা রিসার্চের জন্য লিনাক্সে (Debian/Kali) নিচের টুলগুলো অপরিহার্য:

* **Crunch:** কাস্টম প্যাটার্নের ওয়ার্ডলিস্ট তৈরির জন্য ব্যবহৃত হয়।
* **Rockyou.txt:** ১.৪ কোটিরও বেশি লিক হওয়া পাসওয়ার্ডের একটি বিশাল ডিকশনারি।
* **John the Ripper / Hashcat:** পাসওয়ার্ড হ্যাশ ক্র্যাক করার জন্য সবচেয়ে শক্তিশালী টুল।

### 📊 পাসওয়ার্ড স্ট্রেন্থ টেবিল
| পাসওয়ার্ডের ধরণ | ৪-৮ ক্যারেক্টার | ১০-১২ ক্যারেক্টার | ১৫+ ক্যারেক্টার |
| :--- | :--- | :--- | :--- |
| **শুধু সংখ্যা** | মুহূর্তেই ক্র্যাক হয় | কয়েক সেকেন্ড | কয়েক মিনিট |
| **অক্ষর + সংখ্যা + সিম্বল** | কয়েক ঘণ্টা | কয়েক বছর | বিলিয়ন বছর |

## 💡 পেনটেস্টার টিপস (Best Practices)
1. **Default Passwords:** সর্বদা রাউটার বা ডিভাইসের ডিফল্ট পাসওয়ার্ড (admin/admin) পরিবর্তন করুন।
2. **System Privilege:** উইন্ডোজের ক্ষেত্রে **NT AUTHORITY\SYSTEM** এবং লিনাক্সে **Root** এক্সেস পাওয়া হলো চূড়ান্ত লক্ষ্য।
3. **Legal Lab:** সবসময় VirtualBox বা VMware-এ নিজের ল্যাব বানিয়ে প্র্যাকটিস করুন।

> **পরবর্তী পদক্ষেপ:** আপনি কি **Crunch** ব্যবহার করে একটি কাস্টম ওয়ার্ডলিস্ট তৈরির প্রাকটিক্যাল কমান্ড দেখতে চান? অথবা সরাসরি কোনো সার্ভিসের (যেমন: SSH বা FTP) ওপর ডিকশনারি অ্যাটাক কীভাবে চালাতে হয় তা জানতে চান?

চমৎকার! একজন পেনটেস্টার বা ইথিক্যাল হ্যাকার হিসেবে **Crunch** এবং **Hydra** (SSH/FTP অ্যাটাকের জন্য) সম্পর্কে জানাটা আপনার জন্য অনেক বড় একটা অ্যাসেট হবে। যেহেতু আপনি ডেবিয়ান ১৩ (Debian 13) ব্যবহার করছেন, আপনি এখনই এই টুলগুলো ট্রাই করতে পারেন।

নিচে আমি প্র্যাকটিক্যাল কমান্ডগুলো সাজিয়ে দিচ্ছি:

---

### ১. Crunch ব্যবহার করে কাস্টম ওয়ার্ডলিস্ট তৈরি
**Crunch** দিয়ে আপনি যেকোনো প্যাটার্নের পাসওয়ার্ড লিস্ট তৈরি করতে পারেন।

* **বেসিক সিনট্যাক্স:**
    `crunch <min_len> <max_len> <characters> -o <output_file.txt>`

* **উদাহরণ ১ (শুধু সংখ্যা):** আপনি যদি ৪ থেকে ৬ অক্ষরের শুধু সংখ্যা দিয়ে একটি লিস্ট চান:
    ```bash
    crunch 4 6 0123456789 -o numeric_pass.txt
    ```

* **উদাহরণ ২ (প্যাটার্ন ভিত্তিক):** ধরুন আপনি জানেন টার্গেটের নাম 'yasin' এবং শেষে ৩টি সংখ্যা আছে। `^` দিয়ে চিহ্ন এবং `@` দিয়ে লোয়ারকেস বোঝায়, কিন্তু নির্দিষ্ট প্যাটার্নের জন্য `-t` ব্যবহার করা হয়:
    ```bash
    # এখানে @ মানে ছোট হাতের অক্ষর এবং % মানে সংখ্যা
    crunch 8 8 -t yasin%%% -o yasin_list.txt
    ```
    এটি `yasin000` থেকে `yasin999` পর্যন্ত সব পাসওয়ার্ড তৈরি করে দেবে।

---

### ২. SSH বা FTP সার্ভিসে ডিকশনারি অ্যাটাক (Hydra)
একবার ওয়ার্ডলিস্ট তৈরি হয়ে গেলে, সেটি দিয়ে কোনো সার্ভিসে লগইন করার চেষ্টা করতে **Hydra** ব্যবহার করা হয়।

* **SSH অ্যাটাক কমান্ড:**
    ```bash
    hydra -l user123 -P yasin_list.txt ssh://192.168.1.10
    ```
    * `-l`: টার্গেট ইউজারনেম।
    * `-P`: আপনার তৈরি করা পাসওয়ার্ড লিস্টের পাথ।
    * `ssh://...`: টার্গেট আইপি অ্যাড্রেস।

---

### ৩. পাসওয়ার্ড হ্যাশ ক্র্যাকিং (John the Ripper)
যদি আপনি কোনোভাবে সিস্টেমের পাসওয়ার্ডের হ্যাশ (যেমন লিনাক্সের `/etc/shadow` ফাইল) পেয়ে যান, তবে তা ক্র্যাক করতে **John** ব্যবহার করবেন:

```bash
# rockyou.txt ব্যবহার করে হ্যাশ ক্র্যাক করা
john --wordlist=/usr/share/wordlists/rockyou.txt myhash.txt
```

আপনার ডেবিয়ান মেশিনে এগুলো টেস্ট করার জন্য:
1.  **ইনস্টল করতে:** `sudo apt update && sudo apt install crunch hydra john -y`
2.  **সতর্কতা:** এই কমান্ডগুলো শুধুমাত্র আপনার নিজের তৈরি করা ভার্চুয়াল মেশিন (VM) বা ল্যাব এনভায়রনমেন্টে চালাবেন। অনুমতি ছাড়া অন্য কারো আইপিতে এগুলো চালানো বেআইনি।

**আপনার পরবর্তী আগ্রহ কি?** আমি কি আপনাকে লিনাক্সে এই ল্যাবটি সেটআপ করার জন্য একটি ছোট **গাইড** লিখে দেব, নাকি আপনি অন্য কোনো হ্যাকিং মেথডোলজি (যেমন: Sniffing বা Web Exploit) সম্পর্কে জানতে চান?
